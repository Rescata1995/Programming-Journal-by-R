Probando otras etiquetas!

Etiquetas restantes: "address", "blockquote", "pre", "div".

Etiqueta "address"

La etiqueta correcta para mostrar una dirección es: "address". Se utiliza para aportar información de contacto para el "article" en el que esté trabajando (es decir, en qué lugar se desarrolló el "article" que está trabajando; en este caso, las etiquetas "address" irían dentro de las etiquetas "article" cuando desee dejar claro dicha dirección); o bien, "address" puede ir dentro del "footer" del "body": por lo general se refiere a la dirección de quién está escribiendo el documento HTML, puede ser la dirección de una empresa o de una persona natural; ojo, no es la dirección de dónde, por ejemplo, se desarrollaron los eventos en un "article", no; sino, la dirección de quién está escribiendo el "article" en sí y todo el documento HTML propiamente. En nuestro "footer" verá reflejado con un ejemplo la puesta en práctica de la etiqueta "Address".

Etiqueta "blockquote"

Se utiliza para marcar las citas a otros autores o documentos. Se puede incluir, además, el atributo "cite" para poner un enlace al documento original o fuente. Queda algo así:

Según MDN Web Docs, blockquote -cita en bloque . Crea citas en bloque, marca las citas a otros autores o documentos. Su etiqueta es: "blockquote".

Etiqueta "pre"

Se utiliza para tener código preformateado que necesite ser representado igual que se escribió dentro de HTML (que los saltos de líneas y la identación, por ejemplo, que le demos a nuestra escritura en HTML se refleje igualmente en nuestro navegador). Este recurso se utiliza mucho cuando estamos escribiendo letras de canciones o algún poema. Veamos esto con el siguiente poema:

Aquí (Octavio Paz) 
                    Mis pasos en esta calle
                    Resuenan
                    En otra calle
                    donde
                    Oigo mis pasos
                    Pasar en esta calle
                    donde
                    Sólo es real la niebla.

Etiqueta "div"

Se usa como división del documento, semánticamente no significa nada, es un contenedor genérico que se usa generalmente para dar estilos a través de CSS o para usar algo denominado "delegación de eventos" en JS. Se usa para dividir contenido; todo contenido, con "div", debe tener por separado sus propias etiquetas "div"... proximamente tocaremos más a fondo este tema.

Etiqueta "hr"

"Horizontal rule", esta etiqueta se utiliza para decirle al navegador que vamos a cambiar de tema. "hr", visualmente, se ve como una linea recta al horizonte; sin embargo, dejemos algo claro: "hr" no se utiliza para "dibujar lineas", no; sino, para señalarle al navegador que vamos a hablar de un tema completamente diferente en el próximo articulo o en el próximo párrafo. El concepto propio de la documentación plantea que "hr" representa una rotura de tematica a nivel de párrafos; por ejemplo, una escena que cambia de historia o una transición hacia otro tema.
